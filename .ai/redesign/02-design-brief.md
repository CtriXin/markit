# OpenDesign Restyle Design Brief

## Component Language

- App shell: charcoal `#1f1e1b`, off-black `#171614`, thin borders `#302d28`.
- Sidebar: comment-review panel, dark list rows, small muted meta text, square check toggles.
- Stage: neutral light preview viewport on a dark editor surface; annotation/target outlines use `#2d7dff`.
- Controls: compact rounded rectangles; active tools are dark or blue, not green/beige.
- Bug/detail cards: dark inspector panels with minimal borders; severity remains red for priority only.

## Anti-patterns

- No warm paper background.
- No big marketing hero feeling inside the tool.
- No chunky beige cards.
- No decorative gradients as design proof.

## QA Targets

- session workspace screenshot should read as OpenDesign-like editor.
- bug list/detail should look like product review surface, not generic dashboard.
- all previous E2E behaviors must still pass.
